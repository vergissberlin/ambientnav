Flashe die AmbientNav-Firmware direkt aus dem Browser auf deine ESP32-Boards mit
[ESP Web Tools](https://esphome.github.io/esp-web-tools/).
Keine Treiber oder Toolchain-Installation erforderlich.

:::caution[Reihenfolge beim Flashen beachten]
Flashe immer zuerst das **Rear-Board (hinten)**, danach das Front-Board. Das Front-Board sucht
beim ersten Start per Bluetooth Classic nach dem Rear-Board. Falls die Reihenfolge falsch war,
schalte das Rear-Board ein und flashe das Front-Board erneut.
:::

---

## Browser-Voraussetzungen

| Anforderung | Details |
|---|---|
| Browser | Google Chrome oder Microsoft Edge — nur Desktop |
| Verbindung | USB-Kabel zum ESP32 |
| Protokoll | HTTPS (diese Seite verwendet es bereits) |
| Betriebssystem | Windows, macOS oder Linux |

Safari und alle mobilen Browser unterstützen die Web Serial API nicht und können nicht zum
Flashen verwendet werden.

---

## Schritt 1 — Rear-Board flashen (Slave)

Das Rear-Board steuert die drei Ultraschallsensoren und den hinteren LED-Streifen.

1. Verbinde den **hinteren ESP32** per USB mit dem Computer.
2. Klicke den Button unten und wähle im Browser-Dialog den korrekten seriellen Port aus.
3. Warte ca. 30 Sekunden bis der Flashvorgang abgeschlossen ist.

<EspFlashButton
  manifest="/ambientnav/manifest-rear.json"
  label="⚡ Rear-Board flashen"
/>

Nach dem Flash das Rear-Board trennen und über die 5-V-Versorgung betreiben. Ein langsames
weißes Atmen auf dem hinteren LED-Streifen bestätigt, dass das Board bereit ist.

---

## Schritt 2 — Front-Board flashen (Master)

Das Front-Board übernimmt die BLE-Kommunikation mit dem iPhone und steuert den vorderen LED-Streifen.

1. Verbinde den **vorderen ESP32** per USB mit dem Computer.
2. Klicke den Button unten und wähle den korrekten seriellen Port aus.

<EspFlashButton
  manifest="/ambientnav/manifest-front.json"
  label="⚡ Front-Board flashen"
/>

Nach dem Einschalten sucht das Front-Board per Bluetooth Classic nach dem Rear-Board. Sobald
die Verbindung steht, starten beide LED-Streifen den Ambient-Breathing-Effekt.

---

## Nach dem Flashen

1. **iOS-App öffnen** — auf Verbinden tippen. Die App erkennt das Front-Board per Bluetooth LE automatisch.
2. **Bluetooth-Classic-Link prüfen** — zeigen die Front-LEDs dauerhaftes Rot, beide Boards neu starten,
   Rear zuerst.
3. **Probleme beim Koppeln?** Siehe den [Bluetooth-Setup](/en/bluetooth-setup/)-Leitfaden.

---

## Was wird geflasht?

Jeder Button flasht ein einzelnes zusammengeführtes Binary mit Bootloader, Partitionstabelle,
Boot-Applikation und Firmware — identisch zu `pio run --target upload`.

| Datei | Inhalt |
|---|---|
| `firmware-front.bin` | Bootloader + Partitionen + App (Front / Master) |
| `firmware-rear.bin` | Bootloader + Partitionen + App (Rear / Slave) |

Die installierte Version entspricht immer dem [neuesten GitHub-Release](https://github.com/vergissberlin/ambientnav/releases/latest).
